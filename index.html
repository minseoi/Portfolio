// 01. TECHNICAL CASES
CASE 01

Data-driven Skill / Buff System

테이블 데이터를 기반으로 스킬/버프 구조를 구성하여, 잦은 밸런스 변경에 대응하고 클라이언트와 서버 간 데이터 일관성을 확보하였습니다.

Problem

스킬/버프는 기획 변경과 밸런스 조정이 잦아, 코드 중심 구조로 구현할 경우 간단한 수치 변경 시에도 개발 작업이 필요했습니다.

클라이언트와 서버가 서로 다른 기준으로 스킬/버프를 처리할 경우 데이터 불일치로 인한 버그가 발생할 수 있었습니다.

유사한 효과나 조건을 가진 스킬/버프가 많아 코드 중복이 증가할 가능성이 있었습니다.

Solution

스킬/버프의 수치, 조건, 효과 구성을 데이터 테이블 기반으로 분리하였습니다.

클라이언트와 서버가 동일한 테이블 정의를 기준으로 스킬/버프를 처리하도록 구성하였습니다.

Result

수치 변경 및 효과 조합 변경 시 코드 수정 부담을 줄였습니다.

클라이언트와 서버 간 스킬/버프 처리 기준을 일치시켜 데이터 불일치 가능성을 낮췄습니다.

신규 스킬/버프 추가 시 기존 효과 구성을 재사용할 수 있어 코드 중복을 줄였습니다.

CASE 02

Modular Character System

부위별 장비와 코스튬을 모듈화하여, 장비 슬롯 점유와 코스튬 우선 표시를 지원하는 캐릭터 외형 변경 구조를 구현하였습니다.

Problem

장비와 코스튬이 머리, 상체, 하체 등 여러 부위 파츠로 나뉘어 있어 부위별 장착 상태를 관리해야 했습니다.

일부 장비는 여러 슬롯을 동시에 점유할 수 있어 단순한 1슬롯 1장비 구조로는 처리하기 어려웠습니다.

장비 외형보다 코스튬 외형이 우선 표시되어야 하는 규칙이 필요했습니다.

Solution

부위별 장비 슬롯과 코스튬 슬롯을 분리하여 장착 구조를 구성하였습니다.

장비의 슬롯 점유 정보를 기준으로 장착 가능 여부와 충돌 처리를 구현하였습니다.

Leader Pose와 Skeletal Mesh Merge를 활용하여 부위별 외형 변경을 처리하였습니다.

Result

장비와 코스튬 구조를 공통화하여 신규 파츠 추가 시 작업 비용을 줄였습니다.

장비, 코스튬, 부위별 파츠를 유연하게 조합할 수 있는 캐릭터 외형 시스템을 구성하였습니다.

CASE 03

Dynamic Level Loading

하나의 Persistent Level 안에서 대기실과 전장 Sub Level을 컨테이너 단위로 로드/언로드하는 레벨 로더를 구현하여, 전장 추가와 관리 비용을 줄였습니다.

Problem

공통 대기실과 전장별 맵을 각각 별도의 Persistent Level로 구성할 경우, 전장마다 레벨 생성 및 매핑 작업이 필요해 관리 부담이 증가했습니다.

대기실과 전장 역시 디자인 레벨, 라이팅 레벨, 네비게이션 레벨 등 여러 Sub Level로 구성되어 있어 레벨 관리가 복잡했습니다.

대기실과 전장이 함께 로드되는 상황에서 현재 전장에 맞는 네비게이션 데이터를 적용해야 했습니다.

Solution

하나의 Persistent Level 안에 여러 Sub Level을 배치하고, 관련 Sub Level들을 컨테이너 단위로 묶어 로드/언로드하는 레벨 로더 모듈을 구현하였습니다.

전장별 네비게이션 데이터를 관리하고, 런타임에 현재 전장에 맞는 NavMesh 데이터를 적용하도록 구성하였습니다.

로딩 화면과 연계하여 대기실에서 전장으로 전환되는 흐름을 처리하였습니다.

Result

신규 전장 추가 시 별도의 Persistent Level 생성과 매핑 작업을 줄였습니다.

컨테이너 단위로 전장을 관리하면서도, 컨테이너 내부의 디자인, 라이팅, 네비게이션 Sub Level을 세분화하여 관리할 수 있었습니다.

Persistent Level을 유지한 상태에서 대기실과 전장 간 전환을 자연스럽게 처리할 수 있었습니다.

CASE 04

VR Hand Tracking Wrapper

VR 플랫폼별로 다른 핸드 트래킹 본 구조와 회전값을 공통 인터페이스로 추상화하여, 여러 플랫폼에서 일관된 핸드 트래킹 및 제스처 처리가 가능하도록 구현하였습니다.

Problem

Meta, Pico 등 VR 플랫폼마다 핸드 트래킹에 사용하는 본 구조, 기준 회전값, 컴포넌트 구성이 달라 동일한 핸드 메시를 그대로 사용하기 어려웠습니다.

플랫폼별 차이를 그대로 노출할 경우, 제스처 인식 로직과 애니메이션 처리 로직이 플랫폼마다 분기되어 유지보수 부담이 커질 수 있었습니다.

Solution

플랫폼별 핸드 본 구조와 회전값을 공통 기준에 맞게 보정하는 래퍼 계층을 구현하였습니다.

플랫폼별 입력 및 트래킹 데이터를 공통 인터페이스로 변환하여, 상위 로직에서는 동일한 방식으로 손가락 포즈와 제스처 정보를 사용할 수 있도록 구성하였습니다.

Result

하나의 핸드 메시와 공통 제스처 로직을 여러 VR 플랫폼에서 재사용할 수 있었습니다.

플랫폼별 차이를 래퍼 내부로 숨겨 상위 시스템의 분기 처리를 줄였습니다.

추가 플랫폼 대응 시 래퍼 계층 확장을 통해 즉시 지원할 수 있는 구조를 마련하였습니다.

CASE 05

VR Combat Hit Detection

VR 컨트롤러의 빠른 이동으로 인해 프레임 단위 충돌 감지가 누락되는 문제를 해결하기 위해, 컨트롤러 이동 궤적 기반의 히트 감지 로직을 구현하였습니다.

Problem

VR 컨트롤러는 짧은 시간에 큰 거리를 이동할 수 있어, 단순 Overlap 또는 박스 충돌 감지만으로는 프레임 사이의 충돌을 놓칠 수 있었습니다.

컨트롤러의 미세한 물리적 떨림으로 인해 스윙 방향과 공격 판정이 불안정해질 수 있었습니다.

Solution

Low Pass Filter를 적용하여 컨트롤러 위치 및 방향 데이터의 떨림을 완화하였습니다.

이전 프레임과 현재 프레임의 컨트롤러 위치 및 회전값을 기록하고, 이동 궤적을 기준으로 충돌 판정을 수행하였습니다.

스윙 방향을 계산하여 공격 방향 기반의 후속 전투 로직에 활용할 수 있도록 구성하였습니다.

Result

빠른 컨트롤러 움직임에서도 히트 누락을 줄여 VR 전투의 타격감을 개선하였습니다.

스윙 방향 감지 정확도를 높여 에어본, 오버킬 등 방향 기반 전투 연계 기능을 구현할 수 있었습니다.
